2.1 Design / Traffic Flow

🛠️ Primary/DR Architecture:
Azure East is the primary region for KD Bus site connectivity. Azure South East is the DR region. IPsec VPN configuration and UDRs are pre-staged in both regions so that, in a DR event, workloads in the DR region retain connectivity to all site subnets via the South East FortiGate.
Aspect Description
Route Table Isolation KD-Prod-RouteTable is associated only with East workload subnets; KD-Mel-RouteTable only with SEast workload subnets. Each region routes to sites via its local FortiGate. This prevents asymmetric routing during normal operation.
Why Routes in Both Regions Site subnets are pre-staged in both route tables so that, during a DR event, workloads failed over to SEast retain connectivity to all site subnets via the SEast FortiGate.
DR Failover Method Manual failover (not automatic). DR is invoked by Azure workload failover (ASR/manual), not by this VPN configuration.
DR Scope DR failover testing is not in scope for this CR; DR readiness is achieved by pre-staging configuration only.

11. Rollback Procedure

🚨 Trigger: Execute rollback if tests fail and issue cannot be resolved within change window.
ℹ️ Why Both Regions: Rollback removes both primary (East) and DR (SEast) paths to force reversion to SonicWall and prevent partial failover or asymmetric routing scenarios.

SonicWall Fallback: SonicWall VPN configuration remains unchanged until completion of all batches + 14 days soak period.

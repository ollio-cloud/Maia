7. Test Plan

  • Ping from ATE-TEST (10.160.10.x) to Azure East (10.200.10.10) - expect reply
  • Ping from ATE-TEST to Azure SEast (10.201.10.61) - expect reply
  • RDP to Azure VMs to confirm no asymmetric routing
  • Verify VPN tunnel stats incrementing on FortiGates
  • Verify HTTPS to VPN1, VPN2, EMS1 hostnames shows valid certificate
  • Confirm other KD BUS networks unaffected (no FortiGate tunnels visible)

8. Rollback Procedure

If tests fail: Execute in order (estimated < 5 minutes)
  1. Remove route from KD-Prod-RouteTable (Azure East)
  2. Remove route from KD-Mel-RouteTable (Azure SEast)
  3. Disable/remove ATE-TEST VPN on both FortiGates
  4. Delete IPsec peers in Meraki Dashboard
  5. Verify ATE-TEST connectivity via existing SonicWall tunnels

9. Prerequisites

  • Wildcard certificate *.keolisdowner.com.au obtained
  • DNS A records created and propagated
  • FortiGate firewalls forwarding to Azure backend
  • NSG/Azure Firewall permits FortiGate inside IPs
  • Meraki Dashboard access with Org write permissions
  • PSK shared securely (>32 characters)

1. Overview & Objective

This change establishes IPsec VPN connectivity between Meraki MX appliances and Azure-hosted Fortigate firewalls as a pilot test before full production rollout. The pilot will validate:

  • IPsec tunnel establishment (IKEv2) between Meraki and Fortigate
  • Bidirectional traffic flow (Site → Azure and Azure → Site)
  • Route table configuration in Azure
  • Network tag isolation (test site only)
ℹ️ Pilot Scope: This CR applies ONLY to the ATE-TEST network. Production sites will be migrated via separate CR after successful pilot validation.

7. Fortigate Configuration

⚠️ Security Note: PSK must be >32 characters. DH Group 14 (2048-bit) minimum required.

7.1 Phase 1 (IKE) Configuration

Apply to both Fortigate East and Fortigate South East:

config vpn ipsec phase1-interface edit "ATE-TEST" set interface "port1" set ike-version 2 set keylife 1800 set proposal aes256-sha256 set dhgrp 14 19 set remote-gw [MERAKI-PUBLIC-IP] set psksecret "[PSK > 32 Characters]" next end

7.2 Phase 2 (IPsec) Configuration

Fortigate East (68.218.113.249):

config vpn ipsec phase2-interface edit "ATE-TEST" set phase1name "ATE-TEST" set proposal aes256-sha256 set dhgrp 14 19 set keylifeseconds 3600 set src-addr-type name set dst-addr-type name set src-name "KD-Azure-SYD-Subnet" set dst-name "ATE-TEST-Subnet" next end

Fortigate South East (4.200.139.87):

config vpn ipsec phase2-interface edit "ATE-TEST" set phase1name "ATE-TEST" set proposal aes256-sha256 set dhgrp 14 19 set keylifeseconds 3600 set src-addr-type name set dst-addr-type name set src-name "KD-Azure-MEL-Subnet" set dst-name "ATE-TEST-Subnet" next end

13. Production Rollout (Post-Test Success)

After successful pilot validation, production migration will follow the full transition runbook:

  1. Remove fortigate-test tag restriction from IPsec peers
  2. Create Meraki template with both SonicWall and Fortigate VPN configurations
  3. For each production site:
    • Apply template to site
    • Move Fortigate VPN from loopback to port1 WAN interface
    • Confirm Fortigate tunnel UP
    • Disable SonicWall tunnel
    • Update Azure route tables
    • Execute test checkpoints
  4. After all sites migrated: Update default gateway routes, decommission SonicWall
⚠️ Separate Change Request required for production rollout.

1. Summary

Purpose: Migrate 21 Azure Standard SKU Static public IPs from SonicWall/Sophos NICs to FortiGate NICs, completing the firewall transition. This includes 2 primary outbound NAT IPs (whitelisted by external services), default route (0.0.0.0/0) changes, and 19 inbound service IPs.

Strategy: SE (DR) first for zero-risk validation, then East (production) during maintenance window. Service IPs migrated in phased groups after primary IP + route changes are stable.

7. Test Plan

Per primary IP swap:

  • Outbound SNAT: curl ifconfig.me from FortiGate CLI — confirm correct whitelisted IP
  • External service connectivity: Verify APIs/SaaS still accept traffic from whitelisted IP
  • Azure PIP association: az network public-ip show confirms ipConfiguration points to FortiGate NIC

Per service IP:

  • Inbound reachability: Browser / curl / telnet to service port via public IP
  • FortiGate VIP sessions: diagnose sys session filter dport PORT — sessions visible
  • Traffic logs: execute log filter field dstip 10.200.10.x — action=accept

8. Rollback Procedure

Trigger: Execute if service unreachable after migration and not resolved within 10 minutes.

Primary IP rollback (~30 seconds):

  1. Revert default route (0.0.0.0/0) to previous next-hop
  2. Swap FortiGate primary PIP back to original (ATE-AE-FGT-PIP2 / ATE-ASE-fgt-pip2)
  3. Re-associate outbound IP to original source NIC

Service IP rollback:

  • Per IP: azure-pip-migration-script.ps1 -Rollback -Name "PIP-NAME"
  • Per group: azure-pip-migration-script.ps1 -Rollback -Group "GROUP"
All IPs are Standard SKU Static — IP address is retained when unassigned. Rollback is a simple NIC reassociation (~5 seconds per IP), no IP address change occurs.

9. Prerequisites

  • Azure CLI logged in with permissions to modify NICs, PIPs, and Route Tables in KD-Prod-RG, KD-Prod-RG-asr, and KD-Mel-Temp
  • FortiGate VIP objects configured for all service IPs (destination NAT: public IP → internal service)
  • FortiGate firewall policies permit VIP traffic (inbound) and outbound SNAT
  • Current 0.0.0.0/0 route next-hop recorded for both route tables (needed for rollback)
  • SonicWall configuration unchanged as fallback until migration fully validated

1. Change Summary

Upgrade the KD ConnX HR/Workforce Management system from the current version to ConnX v7.1.0 2955. This upgrade includes application files, database schema updates, IIS configuration changes, ConnX Web Service (CWS) upgrade, and ConnX Automation Process (CAP) reinstallation.

Business Justification: Upgrade to the latest supported version of ConnX to maintain vendor support, access new features, security patches, and ensure compatibility with current infrastructure. SQL Server 2014 is no longer supported by ConnX as of v7.1.

Key Upgrade Details

Category Requirement Details
Current Version ConnX v6.5.0.2016 or greater Minimum prerequisite for v7.1 upgrade
Target Version v7.1.0 2955 Build date: 2026-02-03
SQL Server SQL Server 2016+ SQL Server 2014 no longer supported; recommend SQL Server 2022
MicrOpay Version v9.4 (if applicable) Must match payroll database version
Upgrade Location ConnX Web Server ONLY Do NOT run on individual client machines

6. Rollback Plan

Full Rollback (if upgrade fails critically)

  1. Stop the ConnX website in IIS Manager
  2. Restore ConnX application files from backup (taken at Step 2.8)
  3. Restore ConnX database from backup (taken at Step 2.9)
  4. Restore ConnX Document Datastore database from backup (if applicable)
  5. Reinstall the previous version of ConnX Automation Process (CAP)
  6. Configure CAP database connection
  7. Start the ConnX website and CAP service
  8. Verify the original version loads and functions correctly
Important: Both application file backup AND database backup are required for a full rollback. The upgrader modifies both the files and the database schema. One cannot be restored without the other.

Partial Rollback (CAP only)

  1. Stop the ConnX Automation Process service
  2. Uninstall the new CAP via Control Panel
  3. Reinstall the previous CAP version
  4. Reconfigure database connection
  5. Start the service

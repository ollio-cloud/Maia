1. Change Summary

Deploy a new Azure Virtual Desktop (AVD) session host and implement FSLogix profile containers for improved user profile management. This change migrates existing local user profiles to centralized FSLogix storage, enabling profile roaming across session hosts.

Business Justification: Currently, user profiles are stored locally on a single session host, limiting scalability and creating a single point of failure. FSLogix enables profile portability, faster logons, and better disaster recovery capabilities.

12. Kerberos Troubleshooting Guide NEW SECTION

Issue Symptoms Resolution
No Kerberos ticket klist shows no cifs ticket Verify user is Azure AD joined; run klist purge and re-login
Access denied to file share FSLogix error "Access is denied" Verify RBAC role assignment; allow 15-30 min for propagation
Wrong Kerberos realm Ticket shows unexpected realm Verify Azure AD Kerberos is enabled on storage account
Profile fails to mount Event ID 31 in FSLogix logs Check storage account firewall allows session host subnet
Fallback Option: If Azure AD Kerberos cannot be resolved during the maintenance window, the FSLogix extension can be redeployed with useStorageKey = true to use storage account key authentication. This requires storing the key securely via cmdkey.

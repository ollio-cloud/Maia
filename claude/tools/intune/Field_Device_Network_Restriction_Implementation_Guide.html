1. Executive Summary

This document provides step-by-step implementation guidance for configuring secure field devices using Microsoft Intune and Group Policy. The solution addresses six use cases focused on network restriction, USB control, and secure access patterns.

Key Outcomes

ObjectiveSolution
Disable Wi-FiIntune PowerShell + Scheduled Task
Allow LAN/5G onlyNetwork adapter management
Restrict internet to Azure Bastion + ShareFileWindows Firewall Dynamic Keywords
USB device whitelistingDefender for Endpoint Device Control
USB-to-Serial supportDriver deployment + whitelisting
Secure offline accessBitLocker + Defender + cached credentials
MSP patch accessVPN + firewall rules

Technology Stack

  • Microsoft Intune (MDM)
  • Windows Firewall with Dynamic Keywords
  • Microsoft Defender for Endpoint
  • Group Policy (supplemental)
  • PowerShell automation

Step 1.4: Wi-Fi Disable Script

Location: Intune → Devices → Scripts and remediations → Platform scripts → Add

<#
.SYNOPSIS
    Disables Wi-Fi adapter, allows LAN and Cellular (5G SIM)
.DESCRIPTION
    - Disables all Wi-Fi/Wireless adapters
    - Leaves Ethernet (LAN) enabled
    - Leaves Cellular/WWAN (5G SIM) enabled
    - Creates scheduled task for persistent enforcement
#>

#Requires -RunAsAdministrator

$LogPath = "$env:ProgramData\Intune\Logs"
$TaskName = "Intune-WiFiDisableEnforcement"

try {
    # Get all network adapters
    $AllAdapters = Get-NetAdapter

    # Wi-Fi adapters to DISABLE
    $WifiAdapters = $AllAdapters | Where-Object {
        $_.InterfaceDescription -match 'Wi-Fi|Wireless|WLAN|802\.11|WiFi'
    }

    # Disable Wi-Fi adapters
    foreach ($Adapter in $WifiAdapters) {
        Disable-NetAdapter -Name $Adapter.Name -Confirm:$false
    }

    # Create enforcement scheduled task (runs every 15 minutes)
    # ... (task creation code)

    exit 0
}
catch {
    exit 1
}

5. Use Case 3: USB Device Whitelisting

Step 3.1: Collect Approved Device VID/PIDs

# Get VID/PID of connected USB devices
Get-PnpDevice -Class USB | ForEach-Object {
    if ($_.InstanceId -match 'VID_([0-9A-F]{4})&PID_([0-9A-F]{4})') {
        [PSCustomObject]@{
            Name = $_.FriendlyName
            VID = $Matches[1]
            PID = $Matches[2]
            ForPolicy = "VID_$($Matches[1])&PID_$($Matches[2])"
        }
    }
} | Format-Table -AutoSize

12. Rollback Procedures

Warning: Running the rollback script will remove ALL security restrictions. Use only in emergency situations.

Rollback Commands Summary

# EMERGENCY ROLLBACK - Run with -Confirm switch

# 1. Remove scheduled task
Unregister-ScheduledTask -TaskName "Intune-WiFiDisableEnforcement" -Confirm:$false

# 2. Enable Wi-Fi
Get-NetAdapter | Where-Object { $_.InterfaceDescription -match 'Wi-Fi' } | Enable-NetAdapter

# 3. Remove firewall rules
Get-NetFirewallRule -DisplayName "DynKeyword-*" | Remove-NetFirewallRule

# 4. Remove dynamic keywords
Get-NetFirewallDynamicKeywordAddress | ForEach-Object {
    Remove-NetFirewallDynamicKeywordAddress -Id $_.Id
}

# 5. Reset firewall default
Set-NetFirewallProfile -Profile Domain,Private,Public -DefaultOutboundAction Allow
Important: After running the rollback script, also remove the following from Intune:
  • Network restriction configuration profiles
  • Wi-Fi block settings catalog policies
  • Device Control policies
  • Proactive remediation assignments
